Using Cloud Playout to Stream VOD Videos

In this topic you will learn how to create a collection of VOD videos that can be streamed as a linear channel.

The Cloud Playout module enables broadcasters and media publishers to create a collection of VOD videos and Live events and then stream it as a linear channel and leverage all the features provided by Brightcove Live.

Note: This feature is currently in Limited Availability. Contact your Brightcove Account Manager for more information.

To access the Cloud Playout module, click Cloud Playout in the navigation menu. A list of all Cloud Playout channels that have been created will be displayed (if any).

The page will display the following information:

  • Channel Name - Name of the channel
  • Start Time - Start time for the channel
  • End Time - End time for the channel
  • Looping - Indicates whether the channel will loop
  • Destination - Target destination for the channel
  • Date Created - Date the channel was created
  • Last Updated - Date the channel was last updated
  • Status - Channel status
    • DRAFT - Channel has been created but not activated
    • IDLE - Channel is activated but not scheduled to begin streaming
    • CREATING - Channel is in the process of being activated
    • SCHEDULED - Channel is active and ready to stream
    • STARTING - Channel ready to begin streaming
    • RUNNING - Channel is actively streaming
    • STOPPING - Channel streaming is stopping
    • ERROR - An error occurred while creating or streaming the channel

Creating a channel

To create a new Cloud Playout channel, follow these steps.

  1. Click Cloud Playout in the navigation header.
  2. Click Add New.
  3. Enter the channel Identification details:
    • Name - Name for the channel. The name must have English characters and the following special characters are allowed:   _ . : / = + - 
    • Description - Channel description
    • Image URL - URL to an image file for the channel; meant to be a channel image that could be retrieved as part of the channel's metadata
  4. Enter the channel Schedule details.
    • Loop Playout Content - If On, the program will be played in a loop and the End Date/Time will be enabled. The program can still be modified at any time and the changes will be reflected in the current program and the loops to follow.
    • Ad Insertion - If on, allows ads to be displayed during channel playback. If this option is selected, you can only output to a Brightcove Live Event. See the Enabling Server-Side Ads section below.
    • Start Date/Time - Time when the channel will start playing out. The start time for all assets will be calculated by adding their duration to this start time. This time can be updated only when the channel is not in running state.
    • End Date/Time - If no value is entered and Loop Playout Content is not selected, the stream will end once the playout is complete. If Loop Playout Content is selected and there is no End Date/Time, the playout will be streamed until the channel is stopped. This time can be updated at any time.
  5. Enter the channel Destination details. Note that Cloud Playout supports 2 outputs at the same time. For example, you can output HLS into an S3 Bucket and an RTMP feed to a Brightcove Live Channel or a 3rd party ingestion point.
    • Optimize Display for - Controls the bitrate and dimensions for the output
      • Mobile & Web - If checked, HD renditions
      • Smart TV - If checked, full HD renditions
    • Output Region - The region where the channel will be created. Brightcove recommends selecting a region geographically closest to where the majority of your audience is expected.
    • Output to - Select an output method for the channel
      • Brightcove Live Event - Create or use a previously created Brightcove Live Channel. When creating a new channel, the region of the channel will be optimized based upon the Output Region that was selected.
      • RTMP - Select to output the stream to an RTMP or RTP ingestion point
      • Amazon S3 Bucket - Select to push an HLS output to an Amazon S3 bucket. You will be required to enter Bucket Name, Region, Access Key and Secret Key.
  6. Click Create Channel. The new channel will be created with a status of DRAFT.

Adding videos to a channel

To add videos to a channel, follow these steps.

  1. Click + Add Videos. If you are on the channel list page, click the channel name to open the channel settings page and then click + Add Videos.
  2. Click the checkbox next to the videos you want to add to the channel.
  3. Click Add Videos.
  4. Click Save.

 

When videos are added to a channel, a system-generated playlist will be created and visible in the Media module.

Any changes to the playlist will be reflected in the Cloud Playout channel. Also, it is possible for users to delete the playlist without knowing that it is used by a Cloud Playout channel.

Activating a channel

To enable the channel so that streaming can begin, the channel must be activated. To activate a channel, follow these steps.

  1. Click Activate. If you are on the channel list page, click the channel name to open the channel settings page and then click Activate.

Editing and deleting a channel

To edit a Cloud Playout channel, click the channel name.

To delete a Cloud Playout channel, click the checkbox next to the channel name and then click Delete.

Enabling Server-Side Ads

It is possible to configure a channel so that server-side ads are displayed using Brightcove's Server-Side Ad Insertion technology. When this option is selected, the channel can only be output to a Brightcove Live Event.

 

For complete information on Server-Side Ad Insertion, see the Video Cloud SSAI Overview topic.

Generating the embed code for a Cloud Playout channel

To generate the embed code for a Cloud Playout channel, you will use the Live module. For full details on using the Live module, see Creating and Managing Live Events using the Live Module.

Two types of embed code are available:

  • Standard - iframe embed code. Recommended for most cases. This type places the player inside an <iframe> tag.
  • Advanced - In-page embed code. Allows the player to exist in the HTML page directly, not in an iframe. This offers developers the benefits of ease of accessing the player and associated properties and events.

You also have the ability to get the URL to the published player. This is useful for viewing the event in a browser.

To generate the embed code for a live event, follow these steps:

  1. Open the Live module.
  2. Locate the live event the channel will be streaming to. Typically, the event will be located in the Standby tab and have a name of Channel unless you selected a specific event to stream to when the channel was created.
  3. Click on the event name to open the Control Room page.
  4. Click Publish and Embed > Player Configuration in the left navigation.
    Note: If the channel is configured to enable server-side ads (SSAI), Brightcove Player version 6.18.3 and above must be used. Only version 6.18.3 and above players will appear in the dropdown list.
  5. Click the Player URL to preview the event in a browser or copy the appropriate embed code to use.

Viewing channel output and analytics

While the channel is in a RUNNING state, it is possible to view channel output and analytics. Click on the channel on the Channels page to open the channel details. By default, the channel Output & Analytics will be displayed.

Limitations

The following assumptions and limitations apply.

  • Your account must be enabled for Dynamic Delivery
  • The Cloud Playout module is not yet integrated with user management. Everyone in the account will have access to the Cloud Playout module.
  • The input specification of the Cloud Playout channel is fixed with the following settings:
    • CODEC - AVC
    • FRAME RATE <= 30fps
    • BITRATE <= 20 MBps
  • The Cloud Playout channel shall be created only for the account enabled for dynamic packaging and assumes all the assets holds MP4 asset that can be used against the playlist of cloud playout channel.
  • When the channel is restarted between the start time and end time, the program will not be resumed from where it halted in the previous session. Instead, the program will start from the first asset configured in the playlist.
  • There will be a delayed transition from invalid asset URL to the other, which means there will be black screen for more time than normal.
  • If you are trying to fetch the current EPG for the channel for which scheduled end time is already elapsed and channel is in stopped state, then "EMPTY" EPG will be returned or HTTP Error.
  • Current EPG will be generated based on the scheduled start time and end time and its the difference between the end time and start time is greater than 14 days, then it generates only 14 days from the time time requested to end time or 14 days which ever is earlier.
  • You can also query EPG based on any start time and end time and it matches with the historic data and consolidates with futuristic EPG based on the playlist to frame it within the scheduled start time or end time & max 14 days.
  • Ad insert block will be always enabled for ad avail blanking.
  • Ad insert only uses Splice insert mode and not time code based ingest.
  • To have more accuracy on historic EPG, you may need to use Asset duration greater than 2 mins.
  • Input Looper : if the asset URL is duplicated, then input looper rely on order for switching; if the order is changes during the course of the switching , then subsequent N+1 switch it attempt to create would rely on the order. 
  • Input Looper : if the current active asset of media live channel is deleted from the playlist, then input looper does not know which one to pick as follow the current asset, hence it considers first asset in the playlist as follow.
  • Input Looper : input looper attempts to create schedule action based on various factors every one mins, if the asset duration is less than one min, then there is possibility of delayed switching and causes black screen for more time. ( Still not determined , how frequently the input looper can poll!! , may be 20 sec! considering Ad asset of minimum duration of 30 sec?.)
  • The preview image for the CP channel is created on best effort basis.
  • in case of backdoor recover process directly onto Medialive may cause discrepancies in configuration/state in cloud playout.
  • AD slate provisioned in BC live channel created by CP will be set to default and can be overwritten through API if different ad slate ID to be used during the CP "DRAFT" channel state. 
  • Below default configuration shall be used while creating the BC live from CP. 
    content encryption = none
    DVR = false
    Stream to social media = false
    covert live 2 vod = false
    Enable stream status notification = false
    create recurring SEP = true.
  • If the Video Cloud account is configured for geo-restriction, then Cloud Playout channel should be created in the same region where the content can be accessed based upon the Video Cloud geo-restriction policy.

Electronic Program Guide (EPG) Limitations

  • The EPG is generated on best effort / with closest to accuracy.
  • When EPG is initially constructed from the playlist, there may be a start time error as the CP takes some time to initiate the switching.
  • EPG may not be consistent for every retrieval when the below actions are performed as its constructed dynamically based on the current information it holds. (some scenarios are reorder, add/delete of an asset in a playlist, etc.).
  • If any malfunction happens in the switching , and switch time is not accurate.
  • There could be transit inaccuracy for the future EPG, say some cases like playlist switch, delete current active asset in playlist etc. Consumers should use polling to get the closest , best effort EPG at any time and proactive notification on the changes by the server.
