Managing Playback Rights

In this topic you will learn how to create playback rights.

Playback rights are used to restrict video playback. After playback rights have been created, they can be assigned to videos using the Media module.

To access the Playback Rights administration page, click the ADMIN link and then Playback Rights. A list of playback rights will appear.

Note: To access the Playback Rights page, you must be a Video Cloud Studio user with the Administrator role.
Playback rights with a Status of Inactive cannot be assigned to videos in the Media module.

Creating playback rights

To create playback rights, follow these steps.

  1. Log in to Video Cloud Studio as an Administrator.
  2. In the navigation header, click ADMIN and then Playback Rights. Note that you must be an account administrator to access to the ADMIN menu.
  3. Click New Playback Rights.
  4. Enter a Playback Rights Name.
  5. Checking Allow insecure will allow playback requests to be made without a JSON Web Token (JWT).
    Note: Unchecking this option won't affect the restrictions set for that Playback Right, however, the playback request is going to require a valid JWT which is set programmatically at the player level. For information, see the developer document Using Playback Authorization DRM with the Brightcove Player.
  6. Click Save.

Click a restriction link in the left navigation to add restrictions to a playback right:

  • Geographic
  • Domain
  • Proxy

Editing playback rights

To edit playback rights, follow these steps:

  1. In the navigation header, click ADMIN and then Playback Rights.
  2. Click on a playback right.
  3. Make required changes and click Save.

Activating/Deactivating playback rights

Playback rights cannot be deleted, however, they can be deactivated so they can't be assigned to videos in the Media module. To activate/deactivate playback rights, follow these steps:

  1. In the navigation header, click ADMIN and then Playback Rights.
  2. Click on a playback right.
  3. Click Activate to activate a playback right or Deactivate to deactivate a playback right. Playback rights that are deactivated cannot be assigned to videos in the Media module.

Assigning geographic restrictions

Geographic restrictions allow you to limit video playback based upon country, ZIP code and/or designated market area. To assign geographic restrictions to a playback right, follow these steps:

  1. In the navigation header, click ADMIN and then Playback Rights.
  2. Click on a playback right.
  3. In the left navigation, click Geographic Restrictions.

By default, the Global rule will be set to Allow all. This means, by default, there are no geographic restrictions. Geographic restrictions can be added to prevent viewing based upon:

  • Countries
  • ZIP Codes
  • IP Restrictions
  • Designated Market Area (DMA)

Blacklisting countries

Blacklisting countries can be used to prevent viewing in specific countries.

  1. In the Blacklisted Countries field, select the countries that will be prevented from viewing. As you type, the selection list will be filtered.
  2. Click Save.

Blacklisting zip codes

Blacklisting zip codes can be used to prevent viewing in specific zip codes.

  1. In the Blacklisted Zip Codes field, enter a list of zip codes that will be prevented from viewing. The zip codes should be prefixed with an uppercase, two letter country code and hyphen, for example, US-90045.

It is also possible to upload a .csv file containing blacklisted zip codes. Click Upload CSV to browse and select a file to upload.

Note: Uploading a .csv file will replace the contents of the Blacklisted ZIP codes field.

Blacklisting IPs

Blacklisting IPs can be used to prevent viewing based upon IP address.

  1. In the Blacklisted IPs field, enter a range of IP addresses that will be prevented from viewing. CIDR block notation is also supported.
  2. Click Save.

Blacklisting DMA

A designated market area (DMA), also referred to as a media market, is a region of the United States that is used to define television and radio markets. Blacklisting DMAs can be used to prevent viewing based upon DMA.

  1. In the Blacklisted DMAs field, enter DMA codes that will be prevented from viewing.
  2. Click Save.

Assigning domain restrictions

Domain restrictions allow you to limit video playback based upon domain. To assign domain restrictions to a playback right, follow these steps:

  1. In the navigation header, click ADMIN and then Playback Rights.
  2. Click on a playback right.
  3. In the left navigation, click Domain Restrictions.
  4. In the Blacklisted Domains field, enter the domains that will be prevented from viewing.
  5. In the Whitelisted Domains field, enter the domains that will be allowed to view.
  6. Click Save.

Assigning proxy restrictions

Proxy restrictions allow you to limit video playback based upon certain types of connections. To assign proxy restrictions to a playback right, follow these steps:

  1. In the navigation header, click ADMIN and then Playback Rights.
  2. Click on a playback right.
  3. In the left navigation, click Proxy Restrictions.
  4. Select the proxy types to block .
  5. Click Save.
